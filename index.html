01 — About

I'm a software engineer who likes problems with real edges: systems that have to hold up under load, data that has to stay correct, interfaces that have to make sense to someone in a hurry.

Over the last few years I've worked across the stack — from API design and databases to the small interface details that make a tool feel trustworthy. I care about code that's easy to delete as much as code that's easy to write.

Outside of client and product work, I contribute to a couple of open-source tools and write occasional notes on distributed systems.
